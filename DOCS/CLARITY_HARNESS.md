# 📏 CLARITY_HARNESS.md: 「わかりやすさ」の品質基準

このプロジェクトにおける「わかりやすさ」を、AIが自律的に成果物を評価するための「ハーネス（制動・基準）」として定義します。

| 指標 | 定義 | AIが守るべき具体アクション |
| :--- | :--- | :--- |
| **1. 階層性 (Hierarchy)** | 情報を構造化し、重要度順に整理すること。 | 大見出し・中見出しを使い、結論から先に書く。 |
| **2. 簡潔性 (Conciseness)** | 冗長さを削ぎ落とし、本質のみを伝えること。 | 1つの文章を短くし、箇条書きを活用する。 |
| **3. 視覚的マッピング** | 知識の繋がりを「図」や「表」で可視化すること。 | 複雑な関係性はMarkdown表やMermaid図にする。 |
| **4. 具体性 (Specificity)** | 抽象論を避け、実例や数値、符号を示すこと。 | 将棋なら指し手、コードなら具体的な関数名を示す。 |
